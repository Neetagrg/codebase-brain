10 Documented Failure Modes

Click to explore root causes, fixes, and affected files

Robot Falls on Spawn

Physics

Initial pose mismatch or startup.sh not running

Root Cause: Initial pose doesn't match standing pose

Fix: Ensure world file initial_position matches gait controller standing pose

Files: worlds/ardupilot_humanoid.sdf, scripts/startup.sh

Robot Explodes (Joints Fly Apart)

Physics

Physics timestep too large or joint limits violated

Root Cause: max_step_size > 0.004s

Fix: Keep max_step_size ≤ 0.004s, check joint limits

Files: worlds/ardupilot_humanoid.sdf (physics settings)

Gait Controller Connects But No Movement

Control

Gazebo topics not subscribed (plugin not loaded)

Root Cause: ArduPilotPlugin not loaded

Fix: Run check_config.py to verify topics, restart Gazebo

Files: scripts/check_config.py, worlds/ardupilot_humanoid.sdf

Balance Controller Oscillates

Control

PID gains too high or derivative kick

Root Cause: Kp too high (2.0 in balance_controller vs 0.08 in gait)

Fix: Reduce Kp, add derivative filtering, increase damping

Files: scripts/balance_controller.py, scripts/gait_controller.py

Walking Gait Unstable

Gait

Multiple factors: timing, step length, balance gains

Root Cause: No ZMP stability margin, weak balance gains

Fix: Implement ZMP preview control, add hip_roll joints

Files: scripts/zmp_gait_controller.py, scripts/preview_control.py

MAVLink Connection Timeout

Network

ArduPilot not outputting to UDP:14551

Root Cause: MAVLink output not configured

Fix: In MAVProxy: output add 127.0.0.1:14551

Files: MAVProxy configuration

ZMP Controller Import Error

Config

Hardcoded path doesn't exist

Root Cause: sys.path.insert with /home/neetamis/...

Fix: Use relative imports or os.path.dirname(__file__)

Files: scripts/zmp_gait_controller.py (line 6)

Foot Slipping

Physics

Friction coefficient too low

Root Cause: Foot collision mu < 1.0

Fix: Increase in foot collision (try 1.5-2.0)

Files: worlds/ardupilot_humanoid.sdf (foot collision)

Knee Hyperextension

Control

IK solver allows knee < 0

Root Cause: Joint commands violate limits

Fix: Already clamped in inverse_kinematics.py (line 17)

Files: scripts/inverse_kinematics.py

ArduPilot EKF Not Converging

Config

IMU data quality or coordinate transform wrong

Root Cause: Coordinate transforms incorrect

Fix: Verify AHRS_EKF_TYPE=10, check ArduPilotPlugin transforms

Files: mav.parm, worlds/ardupilot_humanoid.sdf
